# Explore With Me

Explore With Me — приложение для публикации событий, поиска мероприятий, подачи заявок на участие и работы с комментариями.

Проект разделён на микросервисы. Внешние запросы проходят через Gateway, 
а внутреннее взаимодействие между сервисами реализовано через Feign-клиенты.

## Архитектура проекта

Проект состоит из следующих сервисов:

- `gateway-server` — единая точка входа во внешнее API приложения. 
Принимает запросы на `localhost:8080` и маршрутизирует их в нужные сервисы.
- `discovery-server` — Eureka Server. Используется для регистрации и обнаружения сервисов.
- `config-server` — Spring Cloud Config Server. Хранит конфигурации микросервисов.
- `user-service` — сервис пользователей.
- `event-service` — сервис событий, категорий и подборок.
- `request-service` — сервис заявок на участие в событиях.
- `comment-service` — сервис комментариев.
- `stats-service` — сервис статистики просмотров.

Основные сервисы запускаются как Eureka Clients и регистрируются в `discovery-server`.

## Конфигурации сервисов

Конфигурации микросервисов находятся в модуле:
infra/config-server/src/main/resources/config

Через Config Server сервисы получают настройки подключения к базе данных, Eureka и другие параметры.

Бизнес-сервисы используют одну базу данных `main`, но работают в разных схемах:

- `user-service` — схема `ewm_users`
- `event-service` — схема `ewm_events`
- `request-service` — схема `ewm_requests`
- `comment-service` — схема `ewm_comments`

Сервис статистики использует отдельную базу данных статистики.

Общие DTO, исключения, обработчик ошибок, Feign-клиенты и их конфигурация вынесены в модуль:
core/interaction-api

## Взаимодействие сервисов

### user-service

Сервис хранит пользователей.

Используется другими сервисами для:

- проверки существования пользователя;
- получения краткой информации о пользователе;
- получения краткой информации о списке пользователей.

К `user-service` обращаются:

- `event-service`
- `request-service`
- `comment-service`

### event-service

Сервис хранит события, категории и подборки.

Использует:

- `user-service` — для проверки пользователей и получения информации об инициаторах событий;
- `request-service` — для получения количества подтверждённых заявок и изменения статусов заявок;
- `comment-service` — для получения опубликованных комментариев к событиям;
- `stats-service` — для сохранения и получения статистики просмотров.

### request-service

Сервис хранит заявки на участие в событиях.

Использует:

- `user-service` — для проверки пользователя;
- `event-service` — для получения информации о событии.

### comment-service

Сервис хранит комментарии к событиям.

Использует:

- `user-service` — для проверки автора комментария;
- `event-service` — для проверки события.

### stats-service

Сервис хранит статистику просмотров.

Используется `event-service` для:

- сохранения информации о просмотре события;
- получения количества просмотров событий.

## Внутреннее API сервисов

Внутреннее API используется только для взаимодействия между микросервисами. 
Оно не проксируется через Gateway.

### user-service
GET /internal/users/{userId}/exists - проверяет существование пользователя.

GET /internal/users/{userId}/short - возвращает краткую информацию о пользователе.

GET /internal/users/short?ids={ids} - возвращает краткую информацию о списке пользователей.

### event-service
GET /internal/events/{eventId}/published - проверяет, что событие существует и опубликовано.

GET /internal/events/{eventId}/request-info - возвращает информацию о событии, 
необходимую `request-service` для работы с заявками.

### request-service
GET /internal/requests/events/{eventId}/confirmed-count - возвращает количество 
подтверждённых заявок на событие.

GET /internal/requests/confirmed-counts?eventIds={eventIds} - возвращает количество 
подтверждённых заявок для списка событий.

GET /internal/requests/events/{eventId} - возвращает все заявки на участие в событии.

POST /internal/requests/events/{eventId}/status?participantLimit={participantLimit} - 
изменяет статусы заявок на участие в событии.

Внутренний endpoint реализован через `POST`, так как он вызывается между сервисами 
через Feign. Внешнее API при этом остаётся без изменений.

### comment-service
GET /internal/comments/published?eventIds={eventIds} - возвращает опубликованные комментарии для списка событий.

## Запуск и проверка

Базы данных и инфраструктура поднимаются через Docker Compose.

Основные сервисы запускаются через IDEA.

Рекомендуемый порядок запуска:

1. `discovery-server`
2. `config-server`
3. `stats-service`
4. `user-service`
5. `event-service`
6. `request-service`
7. `comment-service`
8. `gateway-server`

Основная коллекция Postman и feature-тесты запускаются через Gateway:
http://localhost:8080

Для проверки отдельной коллекции статистики `stats-service` нужно запустить напрямую 
на порту `9090`:
http://localhost:9090

Например, в конфигурации запуска IDEA можно указать program argument:
--server.port=9090

Порт `9090` требуется только для прямого запуска тестов `stats-service`. 
Основная коллекция приложения и тесты feature работают через Gateway на порту `8080`.

## Реализована фича "Комментарии".

POST /users/{userId}/comments - Зарегистрированный пользователь может оставить комментарий к 
опубликованному событию.

PATCH /users/{userId}/comments/{commentId} - Пользователь может изменить свой комментарий, если он 
не опубликован. Если комментарий был отклонен администратором, он возвращается на рассмотрение.

DELETE /users/{userId}/comments/{commentId} - Пользователь может удалить свой комментарий в любом статусе.

GET /admin/comments - Администратор может сделать выборку комментариев с сортировкой по различным 
параметрам.

PATCH /admin/comments - Администратор может перевести комментарии в опубликованные или отклоненные 
по списку их id, если они находятся в статусе PENDING.

DELETE /admin/comments/{commentId} - Администратор может удалить любой комментарий.

GET /events/{id} - вместе с Событием возвращаются его опубликованные комментарии.

GET /users/{userId}/events/{eventId} - вместе с Событием возвращаются его опубликованные комментарии.

GET /admin/events - вместе с подборкой Событий возвращаются их опубликованные комментарии.
